## Glossary

**Orchestrator** — The "brain" of the AI agent system. It decides which sub-tasks to run and in what order. Alpha has four orchestrators: GENERATION (create tests), EDIT (modify tests), LEARN (ingest knowledge), and COMBINE (merge test cases).

**MCP (Model Context Protocol)** — A standard way for AI agents to connect to external tools and data sources (like a vector store or Salesforce). Think of it as a plugin system for AI.

**SigmaLang** — Testsigma's own XML-based language for writing test cases. Alpha generates test cases in this format.

**RAG (Retrieval-Augmented Generation)** — A technique where an AI looks up relevant information from a database before generating a response. Instead of relying only on what it was trained on, it fetches fresh, relevant context first.

**Vector store** — A special database optimized for storing and searching AI-generated embeddings (numerical representations of text). Used by Alpha via MCP for finding similar test cases or knowledge.

**Knowledge graph** — A database that stores information as a network of connected nodes and relationships (e.g., "Screen A" → "button click" → "Screen B"). Sherlock uses Neo4j as its knowledge graph.

**Neo4j** — A popular graph database used by Sherlock to store the map of how an app's screens and actions relate to each other.

**AX Tree (Accessibility Tree)** — A structured, hierarchical description of every UI element on a screen — its type, label, position, and state. Used by screen readers and, in Sherlock's case, to give the AI a precise understanding of what's on screen.

**Locator** — In browser/mobile testing, a locator is how you identify a specific UI element (e.g., "the button with text 'Submit'"). Getting locators right is critical for reliable test automation.

**Task list controller** — In Atto, this component manages the ordered list of steps the AI agent needs to complete during a test run. It tracks what's done and what's next.

**PageState** — A snapshot of what's currently visible on screen (the DOM, elements, layout). Used by Atto to decide what action to take next.

**ActionEdge** — In a knowledge graph, an edge representing a user action (like a click or swipe) that transitions the app from one state to another.

**HER (Hindsight Experience Replay)** — A machine learning technique where an agent learns from failed attempts by treating the failed outcome as a "goal" it was trying to reach. May appear in auto-healing contexts.

**Circular dependency** — When module A imports from module B, and module B also imports from module A, causing Python to fail to load either one. The fix on Alpha's `fix/copilot-issues` branch addresses this.

**DOM (Document Object Model)** — The structured tree of HTML elements that makes up a web page. Atto reads the DOM to understand what's on screen.

**CDP URL (Chrome DevTools Protocol URL)** — A URL that lets external tools connect directly to a running Chrome browser instance to control it programmatically. Atto added support for this to allow connecting to pre-launched browsers.

**Schema** — In software, a schema is a formal definition of the structure of data — what fields exist, what types they are, and how they relate. Sherlock's v4 work is largely about redesigning these blueprints.
