## Glossary

**Orchestrator** — In Alpha, a top-level AI workflow controller. There are four: GENERATION (creates test cases), EDIT (modifies them), LEARN (learns from examples), and COMBINE (merges test cases). Each orchestrator manages a sequence of AI steps.

**Subagent** — A smaller, specialized AI worker called by an orchestrator to do a specific task (e.g., parse a screen, write a step). Think of the orchestrator as the manager and subagents as the specialists.

**MCP (Model Context Protocol)** — A standard way for AI models to connect to external tools and data sources. Alpha uses MCP servers to connect to a vector store and Salesforce.

**SigmaLang** — Testsigma's custom XML-based language for writing test cases. Alpha generates test cases in this format.

**Locator** — In test automation, a way to identify a specific UI element (button, text field, etc.) on a screen so the test knows what to click or type into. Common types include XPath, CSS selectors, and accessibility IDs.

**Circular Dependency** — When Module A depends on Module B, and Module B also depends on Module A. This creates a loop that prevents the code from loading correctly. The fix usually involves restructuring which module owns certain logic.

**RAG (Retrieval-Augmented Generation)** — A technique where an AI looks up relevant information from a database (the "retrieval" step) before generating a response. This makes answers more accurate and grounded in real data. Alpha's vector store MCP server likely supports this.

**Vector Store** — A special database that stores information as mathematical vectors (lists of numbers representing meaning). It lets you search by *meaning* rather than exact keywords — used in RAG systems.

**WebSocket** — A persistent, two-way connection between a client (browser) and server, allowing real-time updates without the client having to repeatedly ask "are you done yet?" Atto's `checkpoint-ws` branch uses this for live agent status.

**Strategy Pattern** — A software design approach where different algorithms or behaviors are encapsulated separately and can be swapped in and out at runtime. Atto's `feat/agentic-execution` branch uses this to support multiple execution strategies.

**Knowledge Graph** — A database that stores information as a network of connected nodes and relationships (e.g., "Screen A" → "leads to" → "Screen B"). Sherlock builds one of these from screen recordings to map how enterprise apps work.

**Neo4j** — The graph database Sherlock uses to store its knowledge graph. Specialized for storing and querying highly connected data.

**Agentic** — Describes an AI system that can take sequences of actions and make its own decisions to accomplish a goal, rather than just responding to a single prompt. Both Alpha and Atto are moving in this direction.

**Playwright** — An open-source tool for automating web browsers (clicking, typing, navigating). Atto uses it to run web-based tests.

**IST (Indian Standard Time)** — UTC+5:30. The Testsigma team is based in India, so timestamps in this report reflect their working hours.
