Sherlock is undergoing its biggest architectural change yet. The team is designing **v4.0**, which completely changes how Sherlock "reads" a screen recording to understand what a user was doing.

In the old system (v3.1), Sherlock would upload the actual video to an AI model (Gemini) and ask the AI to figure out what was happening — like asking someone to watch a movie and summarise it. This was expensive and slow (up to 8 minutes, up to $0.50 per recording).

In the new v4.0 design, instead of uploading video, Sherlock will read the **DOM** (Document Object Model — the raw structured code that describes every element on a webpage) and the **Accessibility Tree** (a special structured list of every interactive element on screen, including what role it plays, e.g., "button", "text field", "checkbox"). These are text files the browser already produces — no video needed. The AI is then only called for the parts that can't be figured out automatically, like "what was the user *trying* to do in this workflow?" This makes the system much cheaper (~$0.05 per recording) and dramatically faster (20–40 seconds).

## Glossary

- **Alpha** — The AI system that generates test cases. Takes a user's instructions and produces SigmaLang XML test scripts.
- **Atto / Atto Browser Agent** — The AI agent that actually *runs* test cases by controlling a real browser (like a robot using Chrome).
- **Sherlock** — Observability tool that watches screen recordings and builds a map ("knowledge graph") of what users do inside an enterprise app.
- **CDP (Chrome DevTools Protocol)** — A low-level protocol that allows tools to remotely control a Chrome browser. Atto now supports connecting to a Chrome session using its CDP URL, rather than always launching a new one.
- **DOM (Document Object Model)** — The internal tree structure of a web page. Every button, text field, and image is a "node" in this tree. Browsers expose this as structured text, which is much cheaper to send to an AI than a video.
- **AX Tree / Accessibility Tree** — A structured list of all interactive elements on a webpage, produced by the browser. Each entry has a `role` (e.g., "button"), a `name` (e.g., "Submit"), and `states` (e.g., "disabled"). Sherlock v4.0 will use this as its primary input instead of video.
- **MCP (Model Context Protocol)** — A standard way to connect AI models to external tools and data sources (like a vector database or Salesforce). Alpha uses MCP servers to give its AI access to existing test libraries and customer data.
- **Orchestrator** — A high-level coordinator that decides *how* to accomplish a goal, then delegates sub-tasks to specialized "subagents." Alpha has 4 orchestrators: GENERATION, EDIT, LEARN, and COMBINE.
- **Subagent** — A smaller, specialized AI component that handles one specific type of task (e.g., "search the vector store" or "format the output"). Orchestrators call subagents.
- **RAG (Retrieval-Augmented Generation)** — A technique where, before generating an answer, the AI first searches a database for relevant existing information. This makes answers more accurate and specific. Alpha likely uses RAG to find similar existing test cases before generating new ones.
- **Vector Store** — A special database that stores information as mathematical vectors (lists of numbers) so that semantically similar items (e.g., two test cases that test similar things) can be found quickly. Alpha uses a vector store via MCP.
- **SigmaLang** — Testsigma's internal XML-based language for writing test cases. Alpha generates test scripts in this format.
- **DTO (Data Transfer Object)** — A simple container class used to move data between layers of an application (e.g., from the API router to the service layer). No business logic — just data.
- **HER (Hierarchical Episode Representation)** — A term from Sherlock's architecture referring to how user workflows are structured: goals are broken into episodes, episodes into steps.
- **PageState** — A snapshot of what a web page looks like at a specific moment (its DOM + visual state). Sherlock tracks sequences of PageStates to understand what a user did.
- **Neo4j** — A "graph database" — unlike a regular database (rows and columns), it stores data as nodes (things) and edges (relationships between things). Sherlock stores its knowledge graph of app workflows in Neo4j.
- **Gemini 2.5 Flash** — Google's AI model used by Sherlock (similar to how Alpha uses Claude). "Flash" refers to a faster, cheaper variant.
- **pHash (Perceptual Hash)** — A fingerprint of an image based on its visual content. Two screenshots that look nearly identical will have similar pHashes. Sherlock v4.0 uses pHash to check if two screens are the same state without calling an AI model.
- **Qdrant** — A vector similarity search engine (similar to a vector store). Sherlock v4.0 uses Qdrant to find screens and elements it has seen before using embedding similarity.
- **feat / fix branch prefix** — Branch naming conventions. `feat/` means a new feature is being built on that branch; `fix/` means a bug is being fixed. `poc` means "proof of concept" — an exploratory experiment to see if an idea works.
- **CI/CD (Continuous Integration / Continuous Deployment)** — Automated pipelines that run tests and publish software whenever code is merged. Atto added a `release.yml` GitHub Actions workflow today.
- **FastAPI** — A Python web framework used by both Alpha and Atto to expose their functionality as HTTP APIs (web endpoints you can call programmatically).
