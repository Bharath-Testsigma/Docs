## Glossary

**Locator** — A way of identifying a specific UI element on a screen (like a button or text field) so an automated test can interact with it. Examples: XPath, CSS selector, resource ID. If a locator breaks (e.g., the app UI changes), the test fails.

**Circular dependency** — A Python import error where module A needs module B, and module B also needs module A. Python can't resolve this loop and crashes at startup. Fixed by restructuring imports.

**MCP (Model Context Protocol)** — A standard way for AI agents to connect to external tools and data sources (like a vector store or Salesforce). Think of it like a plugin system for AI.

**Orchestrator** — A component that coordinates multiple AI agents or steps. In Alpha, the GENERATION orchestrator decides what sub-tasks to run when creating a test case. It's like a project manager for AI tasks.

**RAG (Retrieval-Augmented Generation)** — A technique where an AI, instead of relying only on its training, first retrieves relevant documents from a database and uses those to inform its answer. Used in Alpha to look up existing test cases before generating new ones.

**Vector store** — A specialized database that stores information as mathematical vectors (lists of numbers), which lets you search by *meaning* rather than exact keywords. "Find test cases similar to this one" is a vector search.

**SigmaLang** — Testsigma's internal XML-based language for defining test cases. Alpha generates test cases in this format.

**Strategy pattern** — A software design pattern where you define a family of algorithms (strategies), and the program can pick which one to use at runtime. In Atto, this means the execution logic can switch between web and mobile strategies cleanly.

**System prompt** — Instructions given to an AI model at the very beginning of a conversation that define its role, rules, and behavior. For example: "You are a mobile test executor. Only use actions that are available in the mobile action set."

**WebSocket** — A real-time, two-way communication channel between a server and a browser/client. Unlike regular HTTP (which is request → response), WebSockets stay open and let the server push updates instantly. Used in Atto's `checkpoint-ws` branch to stream live test status.

**Knowledge graph** — A structured network that stores entities (like "Login Screen") and the relationships between them (like "leads to → Dashboard"). Sherlock builds one from screen recordings to map out how an enterprise app works.

**Neo4j** — A graph database designed to store and query knowledge graphs efficiently. Used by Sherlock.

**Gemini 2.5 Flash** — Google's fast, capable AI model. Used by Sherlock to analyze screen recordings.

**Portkey** — A platform for managing, versioning, and A/B testing LLM prompts across services. Alpha uses it to store prompts centrally so they can be updated without code changes.

**Confluence** — Atlassian's wiki/documentation tool. Alpha has a Confluence integration that lets it read team documentation as context when generating tests.
