## Glossary

**Agentic execution** — An AI mode where the system decides its own next steps based on the current situation, rather than following a pre-written script. More flexible, but harder to predict.

**CDP (Chrome DevTools Protocol)** — A protocol built into Chrome/Chromium that allows external tools to control the browser, inspect network requests, and debug JavaScript. Atto can use this to connect to already-running browsers.

**Circular dependency** — A bug where Module A imports Module B and Module B imports Module A, causing both to get stuck waiting for the other to load first. Python raises an error when this happens.

**Copilot** — The AI-powered assistant feature in Alpha that helps users write test cases automatically, by watching what they do or by generating tests from natural language descriptions.

**Knowledge graph** — A database that stores information as a network of connected nodes and relationships. In Sherlock's case, each "node" is a screen or action in an app, and the edges (connections) represent how users move between them.

**Locator** — In test automation, a locator is how you identify a button, text field, or other UI element on screen. Examples: finding by ID, by XPath, by accessibility label. Having multiple locator strategies makes tests more resilient to UI changes.

**LLM (Large Language Model)** — An AI model trained on large amounts of text that can understand and generate human-like language. Claude (used in Alpha) and Gemini (used in Sherlock) are examples.

**MCP (Model Context Protocol)** — A standard way for AI models to connect to external tools and data sources. Alpha uses MCP servers to connect to a vector store (for memory/search) and Salesforce (for customer data).

**Neo4j** — A graph database (a database optimized for storing connected data). Sherlock uses this to store its knowledge graph of app workflows.

**Orchestrator** — A component that coordinates multiple AI agents or steps. In Alpha, there are 4 orchestrators (GENERATION, EDIT, LEARN, COMBINE) each handling a different type of task.

**Playwright** — A popular open-source library for automating web browsers (click buttons, fill forms, navigate pages). Atto uses this to actually run web tests.

**RAG (Retrieval-Augmented Generation)** — A technique where an AI looks up relevant information from a database before generating a response, so its answers are grounded in actual data rather than just its training. Alpha's vector store MCP enables this.

**SigmaLang** — Testsigma's custom XML-based language for defining test cases. Alpha generates test cases in this format so they can be imported and run by the Testsigma test platform.

**Sprint** — A fixed time period (usually 1–2 weeks) in which a software team commits to completing a set of tasks. Branch names like `feat/atto-sprint-6` indicate this is feature work for Sprint 6.

**UUID (Universally Unique Identifier)** — A 128-bit random ID used to uniquely identify something (like a test task) so there's no ambiguity between systems.

**Vector store** — A special database that stores information as mathematical vectors (arrays of numbers), which allows AI systems to search by meaning rather than exact keywords. Alpha uses this to remember past test generation context.

**WebSocket** — A communication protocol that keeps a persistent, two-way connection open between a client and server, enabling real-time updates (unlike regular HTTP which is request-response only).
